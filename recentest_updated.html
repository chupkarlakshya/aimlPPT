Understanding Supervised and Unsupervised Learning

Linear Regression

Predicting continuous outcomes by fitting the best straight line through data.

Understanding
Linear
Regression

Linear regression models the relationship between two variables using a straight line, emphasizing how one variable can predict another through their correlation, aiding in forecasting and data analysis.

SIMPLE LINEAR REGRESSION

Understanding
Linear
Regression
Equation

The linear regression equation, y = mx + b, represents the relationship between the dependent variable (y) and independent variable (x). Here, m is the slope, and b is the y-intercept.

ŷ = m · x + b
📐

m (slope)

Rate of change — how much y increases per unit of x.

🎯

b (intercept)

Value of y when x = 0. Shifts the line vertically.

📊

ŷ (y-hat)

The predicted value — our model's best estimate.

⚡

MSE Cost

J = (1/n)·Σ(yᵢ − ŷᵢ)² — minimized by gradient descent.

Types of Linear
Regression
Explained

Simple

One input variable predicts the outcome.

Multiple

Multiple input variables predict the outcome.

Simple LR: ŷ = m₁x₁ + b  (one predictor)
Multiple LR: ŷ = m₁x₁ + m₂x₂ + … + mₙxₙ + b
Assumptions: Linearity, independence, homoscedasticity, normality of residuals.

Adjust parameters and observe how the regression line changes

Slope controls steepness. Noise adds scatter. Try slope=0 to see a flat line!

Assumptions & Residual Analysis

What must be true for linear regression to work well

📏

Linearity

The relationship between X and Y must be linear. Verified via scatter plot or residual vs. fitted plot.

🔀

Independence

Observations must be independent of each other. Violated in time-series (autocorrelation). Use Durbin-Watson test.

🎯

Homoscedasticity

Variance of residuals should be constant. Heteroscedasticity = fan-shaped residual plot. Fix: log transform Y.

🔔

Normality

Residuals should be normally distributed. Check with Q-Q plot. Important for confidence intervals and p-values.

Measuring
Regression
Performance

Unlike classification, regression quality is measured by how close predictions are to actual values — using distance-based metrics like R², RMSE, and MAE.

R² (R-squared):
Proportion of variance explained. R²=1 is perfect, R²=0 means the model is no better than predicting the mean.
RMSE (Root Mean Squared Error):
√(mean of squared residuals). Same units as Y. Penalises large errors heavily.
MAE (Mean Absolute Error):
Mean of |actual − predicted|. More robust to outliers than RMSE. Easier to interpret.

Basics of Classification

Assigning inputs into categories — the backbone of spam filters, medical diagnoses, and more.

Understanding
Classification in
Machine
Learning

Classification is the process of assigning inputs into distinct classes or categories. This technique is crucial in decision-making systems, enabling applications like spam filtering and image recognition to function effectively.

CLASSIFICATION EXAMPLE

Types of
Classification in
Machine Learning

Binary

Two classes: spam and not spam.

Multi-class

More than two classes such as fruits.

Binary Classification:
Only 2 outputs. Model outputs a probability → threshold → class. Example: Spam vs Not Spam.
Multi-Class Classification:
3+ possible classes. Strategies: one-vs-rest, softmax output. Example: Digit recognition (0–9).
Multi-Label:
One input can belong to multiple classes simultaneously. Example: a photo tagged as both "beach" and "sunset".

Decision Boundaries Explained

How classifiers draw the line between classes

📏 Linear Boundary
Logistic Regression and SVM (linear kernel) draw a straight line in 2D (or hyperplane in higher dimensions) to separate classes. Works perfectly for linearly separable data.
🌀 Non-Linear Boundary
Decision Trees, KNN, and kernel SVMs create curved or jagged boundaries that can capture complex class shapes — but risk overfitting if not regularised properly.
🎯 Probabilistic Output
Rather than hard class labels, many models output a probability score. Threshold (default 0.5) can be tuned to balance precision vs recall based on the cost of each error type.

Case Study:
Email Spam
Filter

A spam filter is one of the oldest and most successful applications of classification. It shows every concept in action: features, training, threshold tuning, and evaluation.

Step 1 — Features: Word frequencies, sender reputation, link count, caps ratio, known spam phrases.
Step 2 — Label: Each training email is labelled "spam" (1) or "not spam" (0) by humans.
Step 3 — Train: Naive Bayes or Logistic Regression learns which feature combinations predict spam.
Step 4 — Threshold: If P(spam) ≥ 0.7, move to junk. Lower threshold = more spam caught, more false positives.

Classification Techniques and Applications

From logistic regression to KNN and decision trees — practical tools that power real-world AI.

Understanding
Logistic
Regression

Logistic Regression is a powerful classification method that predicts probabilities. It effectively determines the likelihood of an outcome, making it crucial in various fields, including healthcare for predicting disease presence.

P(y=1) = σ(w·x + b) = 1/(1+e⁻ᶻ)
  • Uses the sigmoid function to squash any value into probability [0,1].
  • If P(y=1) ≥ 0.5 → class 1, else → class 0.
  • Weights learned by maximizing Log-Likelihood via gradient descent.
  • Highly interpretable — coefficients show log-odds effect of each feature.

📉 Interactive: Sigmoid Curve Explorer

See how weight and bias shape the probability curve

Large w = steep curve. Bias shifts curve left/right. Threshold sets the decision cutoff.

K-Nearest
Neighbors
Deep Dive

KNN is a lazy learner — it stores all training data and only computes at prediction time. Simple, powerful, but slow for large datasets.

Algorithm: (1) Compute distance to all training points → (2) Find K nearest → (3) Majority class vote wins.
Distance Metrics: Euclidean (straight line), Manhattan (city block), Minkowski (generalised). Scale features first!
Choosing K: Small K = sensitive, jagged boundary. Large K = smooth but may misclassify. Use cross-validation to find the best K.
Limitation: O(n) prediction cost — slow at scale. Fails in high dimensions (curse of dimensionality). Not suitable for real-time with millions of records.

K-Nearest Neighbors & Decision Trees

Overview of classification techniques explained

K-Nearest Neighbors
KNN classifies data points based on their nearest neighbors in the dataset, making it effective for simple classification tasks and intuitive to understand. Store data → Compute distances → Vote.
Decision Trees
Decision Trees utilize a tree-like model of decisions, where each node represents a feature, leading to branches and outcomes, making it a powerful tool for classification problems.
Strengths & Weaknesses
KNN is straightforward and easy to implement, while Decision Trees can handle complex data but may lead to overfitting if not properly pruned and managed.

Gradient Descent

The optimization engine behind ML

  • Imagine a hilly landscape — GD finds the lowest valley (minimum loss) by taking steps downhill.
  • Learning rate (α): controls step size. Too large = overshoot. Too small = very slow convergence.
  • Update rule: θ = θ − α · ∂J/∂θ — move in the direction of steepest descent.
  • Converges when the gradient approaches zero — parameters stop changing significantly.
  • Variants: Batch GD (all data), Stochastic GD (one sample), Mini-batch GD (small groups — standard practice).
θ_new = θ_old − α · gradient
Why it matters:
Every ML model from linear regression to deep neural networks is trained by some variant of gradient descent. Understanding it is fundamental to understanding how machines "learn."

Evaluation Metrics

Understanding Key Performance Indicators

🎯 Accuracy
Accuracy measures the overall correctness of a model, indicating the proportion of true results among total cases evaluated, serving as a fundamental metric in model assessment.
🔍 Precision
Precision reflects the accuracy of positive predictions made by the model, showing how many of the predicted positive cases were actually positive, crucial for applications needing high reliability.
🕵️ Recall
Recall evaluates the model's ability to identify all relevant instances, revealing how many actual positive cases were correctly predicted, highlighting its effectiveness in detecting positive outcomes.

Bias–Variance Tradeoff

The fundamental tension every ML practitioner must manage

⬆️ High Bias (Underfitting)
Model is too simple. Misses important patterns. High training error AND high test error. Fix: more complex model, more features, less regularisation.
✅ The Sweet Spot
Model captures real patterns without memorising noise. Low training error, low test error. Achieved through cross-validation, proper regularisation, and enough data.
⬇️ High Variance (Overfitting)
Model is too complex. Memorises training data including noise. Very low training error but high test error. Fix: regularisation (L1/L2), dropout, more training data, pruning.

Overfitting & Underfitting

The core tension in machine learning

📉 Underfitting
Model too simple — high bias. Cannot capture training patterns. High error on both training and test data. Fix: more features, use a more complex model.
✅ Good Fit
Balances bias and variance. Learns patterns without memorizing noise. Generalises well to unseen data. This is the target we aim for through proper training and validation.
📈 Overfitting
Model too complex — high variance. Memorizes training data including noise. Very low training error but high test error. Fix: regularization, dropout, more data, pruning.

Random Forests & Ensemble Methods

The wisdom of the crowd applied to ML

🌲 Bagging
Train many decision trees on random subsets of data, then average their predictions. Reduces variance dramatically compared to a single tree.
🗳️ Majority Vote
For classification, the class voted by the most trees wins. For regression, predictions are averaged. Each tree also sees a random subset of features at each split — further reducing correlation.
🚀 Boosting
Sequentially build trees where each corrects errors of the previous. Powerful algorithms: XGBoost, LightGBM, CatBoost — dominate Kaggle competitions.

Artificial Neural Networks

Inspired by the brain — layers of interconnected neurons

  • Input Layer: receives raw features — pixels, words, numbers.
  • Hidden Layers: transform data into progressively abstract representations.
  • Output Layer: produces the final prediction (class probabilities or a value).
  • Activation Functions: ReLU, Sigmoid, Tanh — introduce non-linearity to model complex patterns.
  • Backpropagation: compute error at output, propagate backwards using chain rule to update each weight.
output = activation(W · x + b)

Cross-Validation & Model Selection

Never evaluate your model on data it learned from

  • Holdout Split: 80% training / 20% test. Simple but dependent on the random split.
  • k-Fold CV: split into k parts; train on k-1, test on 1, repeat k times. More robust estimate.
  • Stratified k-Fold: preserves class distribution in each fold — essential for imbalanced data.
  • Validation Set: a third split for hyperparameter tuning — keeps test set truly unseen.
k-Fold Example (k=5):
Fold 1: [TEST][train][train][train][train]
Fold 2: [train][TEST][train][train][train]
Fold 3: [train][train][TEST][train][train]
Fold 4: [train][train][train][TEST][train]
Fold 5: [train][train][train][train][TEST]

Final score = mean of 5 test results

Support Vector Machines (SVM)

Maximum margin classification

📐 Hyperplane
The decision boundary that separates classes in feature space. SVM finds the hyperplane that maximises the margin between the two classes — best generalization.
📍 Support Vectors
The data points closest to the boundary — they define it. Only these points influence where the hyperplane is placed, making SVM memory-efficient.
🔮 Kernel Trick
Transforms data into higher dimensions to handle non-linear boundaries. Popular kernels: RBF (radial basis), Polynomial. Use when classes aren't linearly separable.
28

ROC Curve & AUC Score

Evaluate a classifier across all decision thresholds

  • ROC Curve: plots True Positive Rate vs False Positive Rate as we vary the threshold from 0 to 1.
  • AUC = 1.0: perfect classifier — always predicts correctly regardless of threshold.
  • AUC = 0.5: random classifier — as good as flipping a coin. No predictive power.
  • AUC > 0.8: good model. AUC > 0.9: excellent. Best metric for comparing multiple models.
Threshold Trade-off:

Lower threshold → more positives predicted → higher Recall, lower Precision.

Higher threshold → fewer positives predicted → higher Precision, lower Recall.

Choose based on whether FP or FN is more costly for your problem domain.

Ethics in Machine Learning

With great predictive power comes great responsibility

  • Bias in Data: biased training data produces biased models. The COMPAS recidivism tool was found to be racially biased against Black defendants.
  • Black-box Problem: deep models are not interpretable — hard to explain why a specific decision was made to regulators or patients.
  • Privacy: models trained on personal data can leak information about individuals — differential privacy helps.
  • Job Displacement: automation of cognitive tasks — requires societal adaptation and responsible deployment strategies.
Responsible AI Principles:

✅ Transparency — explain decisions
✅ Fairness — test across demographic groups
✅ Accountability — humans stay in the loop
✅ Privacy — minimize data collection
✅ Safety — robust to adversarial attacks and edge cases
